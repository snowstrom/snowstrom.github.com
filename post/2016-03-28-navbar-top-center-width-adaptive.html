固定定位相关

固定定位是一个不太常用的css属性，当然，这里说的不太常用是相对于相对定位和绝对定位来说的，当需要将导航条固定在页面的顶部不随页面滚动的时候，可以使用这个固定定位，当然，也可以使用固定定位将页脚固定在页面的底部，原理是一样的。

下面总结一下使用固定定位时遇到的问题，其实这些问题很简单，只是初次使用的时候不太熟练罢了：

1、固定定位和水平居中问题：
	boby{
		margin:0;
		padding:0;
		width:100%;
		overflow:scroll;
	}

	.navbar{
		width:980px;
		height:50px;
		position:fixed;
		left:50%;
		top:0;
		margin-left:-490px;
	}

如上述代码所示，实现了将导航条固定于页面顶部，并实现了居中效果，这里利用了负边距的技巧，就不多说了，大家懂得！给body元素设置overflow属性，是为了当浏览器窗口变窄时，浏览器出现滚动条，以便友好显示过宽的导航条

但是，有一个问题，当使浏览器窗口变窄时，并没有出现滚动条，有一部分导航条的内容是看不到的，这说明body的overflow属性并没有起作用，为什么呢，想一下，哦，注意固定定位也是脱离文档流的，这使得body并“没有”包含导航条，所以导航条的太宽时，也不能触发body的滚动条。

怎么办呢，我的目的是，当浏览器宽度变窄的时候，可以看到整个导航条，那就要使用响应式宽度—百分数啦，使用100%来使得导航条的宽度随浏览器宽度并，但是，当浏览器宽度足够宽的时候，就要固定宽度，所以，使用一个max-width属性好了

对于居中的问题，因导航条的宽度不定，所以上述的负边距方法就失效了，这是使用了同时设置left和right为0的方法，代码如下：

	.navbar{
		max-width: 980px;
        width: 100%;
        height: 80px;
        margin: 0 auto;
        position: fixed;
        top: 0;
        left: 0;
        right: 0;
	}

这样就解决问题了，效果非常好！
